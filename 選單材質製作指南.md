# 日出伺服器 zMenu 材質包製作指南

本文件對應 zMenu 選單中所有已設定 `model-id`（CustomModelData）的物品，供製作自訂材質包時參考。

---

## 一、編號規則

| 範圍 | 選單檔案 | 說明 |
|------|----------|------|
| 10001–10008 | `main.yml` | 主選單 |
| 10009–10014 | `general.yml` | 基礎功能 |
| 10015–10018 | `eco_system.yml` | 經濟金融 |
| 10019–10024 | `home_system.yml` | 住宅系統 |
| 10025–10035 | `tp_system.yml` | 傳送系統 |
| 10036–10039 | `land_system.yml` | 領地系統 |
| 10040–10043 | `chain.yml` | 連鎖採集 |

zMenu 設定欄位為 `model-id`，等同於遊戲內的 **CustomModelData** 數值。

---

## 二、材質包目錄結構

建議的材質包資料夾結構如下：

```
日出選單材質包/
├── pack.mcmeta
└── assets/
    └── minecraft/
        ├── textures/
        │   └── item/
        │       └── menu/              ← 所有自訂貼圖放這裡
        │           ├── tp_system.png
        │           ├── land_system.png
        │           └── ...
        ├── models/
        │   └── item/
        │       ├── menu/              ← 自訂模型 JSON
        │       │   ├── tp_system.json
        │       │   └── ...
        │       ├── compass.json       ← 覆寫原版物品（含 overrides）
        │       ├── oak_door.json
        │       └── ...
        └── items/                     ← 1.21.4+ 新版格式（見下方說明）
            ├── compass.json
            └── ...
```

## 三、pack.mcmeta

```json
{
  "pack": {
    "pack_format": 34,
    "description": "日出伺服器 zMenu 選單材質包"
  }
}
```

> **pack_format 對照**（依伺服器版本調整）：
> - 1.20.5–1.20.6 → `32`
> - 1.21–1.21.1 → `34`
> - 1.21.2–1.21.3 → `42`
> - 1.21.4+ → `46`（需使用新版 `items/` 格式）

## 四、製作流程（1.20.x – 1.21.3）

### 步驟 1：準備貼圖

- 格式：PNG，建議 **16×16** 或 **32×32** 像素
- 放置路徑：`assets/minecraft/textures/item/menu/<名稱>.png`

### 步驟 2：建立自訂模型

範例：`assets/minecraft/models/item/menu/tp_system.json`

```json
{
  "parent": "minecraft:item/generated",
  "textures": {
    "layer0": "minecraft:item/menu/tp_system"
  }
}
```

### 步驟 3：在原版物品模型中加入 overrides

當多個選單按鈕共用同一種 `material` 時，需在對應的原版物品 JSON 中加入 `overrides`。

範例：`assets/minecraft/models/item/compass.json`

```json
{
  "parent": "minecraft:item/generated",
  "textures": {
    "layer0": "minecraft:item/compass_16"
  },
  "overrides": [
    {
      "predicate": { "custom_model_data": 10001 },
      "model": "minecraft:item/menu/tp_system"
    },
    {
      "predicate": { "custom_model_data": 10004 },
      "model": "minecraft:item/menu/external_links"
    },
    {
      "predicate": { "custom_model_data": 10038 },
      "model": "minecraft:item/menu/res_create"
    }
  ]
}
```

### 步驟 4：打包與測試

1. 將整個資料夾壓縮為 `.zip`
2. 放入伺服器 `resourcepacks/` 或玩家 `.minecraft/resourcepacks/`
3. 伺服器執行 `/zm reload` 重新載入選單
4. 開啟選單確認外觀是否正確

---

## 五、製作流程（1.21.4+ 新版格式）

1.21.4 起，物品模型改由 `assets/minecraft/items/<material>.json` 定義。

範例：`assets/minecraft/items/compass.json`

```json
{
  "model": {
    "type": "range_dispatch",
    "property": "custom_model_data",
    "entries": [
      {
        "threshold": 10001,
        "model": {
          "type": "model",
          "model": "minecraft:item/menu/tp_system"
        }
      },
      {
        "threshold": 10004,
        "model": {
          "type": "model",
          "model": "minecraft:item/menu/external_links"
        }
      },
      {
        "threshold": 10038,
        "model": {
          "type": "model",
          "model": "minecraft:item/menu/res_create"
        }
      }
    ],
    "fallback": {
      "type": "model",
      "model": "minecraft:item/compass"
    }
  }
}
```

## 七、需覆寫 overrides 的物品清單

以下 Material 被多個按鈕共用，**必須**在同一個 JSON 檔中列出所有對應的 `custom_model_data`：

| Material | 對應 model-id | 需編輯的 JSON 檔 |
|----------|---------------|------------------|
| `COMPASS` | 10001, 10004, 10038 | `models/item/compass.json` |
| `GOLD_INGOT` | 10003, 10016 | `models/item/gold_ingot.json` |
| `COMMAND_BLOCK` | 10008, 10037 | `models/item/command_block.json` |
| `OAK_DOOR` | 10009, 10015, 10019, 10025, 10036, 10040 | `models/item/oak_door.json` |
| `ELYTRA` | 10010, 10035 | `models/item/elytra.json` |
| `LODESTONE_COMPASS` | 10021, 10026 | `models/item/lodestone_compass.json` |
| `ENDER_PEARL` | 10022, 10032 | `models/item/ender_pearl.json` |
| `BARRIER` | 10023, 10039 | `models/item/barrier.json` |
| `PLAYER_HEAD` | 10027, 10028 | `models/item/player_head.json` |

其餘 Material 各只有一個 model-id，只需在對應的單一物品 JSON 中加入一條 override 即可。

## 八、zMenu 設定範例

選單 YAML 中的寫法如下（已套用至所有選單檔）：

```yaml
btn-10:
  slot: 10
  item:
    material: COMPASS
    model-id: 10001
    name: '&6 [1] 傳送系統'
    lore:
      - '&8傳送與移動'
```

zMenu 也支援以下等效寫法：`customModelData`、`customModelId`、`modelId`。

## 九、除錯技巧

1. **物品仍顯示原版外觀**
   - 確認材質包已啟用（F3+T 重新載入資源）
   - 確認 `custom_model_data` 數值與 `model-id` 完全一致
   - 確認 JSON 語法正確（可用 [jsonlint.com](https://jsonlint.com) 驗證）

2. **部分按鈕正常、部分異常**
   - 檢查共用 Material 的 overrides 是否遺漏某個 model-id
   - 檢查模型 JSON 中的貼圖路徑是否正確

3. **玩家頭顱類物品無效**
   - `PLAYER_HEAD` 與 `MOB_WANDERING_TRADER_FACE` 需特別處理
   - 確認 zMenu 版本支援該 Material 的 model-id

4. **修改選單後**
   - 執行 `/zm reload` 重新載入 zMenu 設定

## 十一、製作及操作方式

1. 將 `日出伺服器專用材質包/assets/minecraft/textures/item/menu/` 內的 PNG 替換成你的自訂貼圖（檔名勿改）
2. 將修改後的檔案commit後推回GitHub (請勿更動其他非材質包資料夾下的檔案，避免造成問題)
3. 等待GitHub action 自動打包完成之後至release下載最新版材質包
4. 放入伺服器或玩家的 `resourcepacks/` 並啟用
5. 將 `MENU(已套用CustomModelData)` 下新版的menu檔案更新至伺服器
6. 執行 `/zm reload`
7. 測試是否正常
